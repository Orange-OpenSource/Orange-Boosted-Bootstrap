## Colors

Content color utility classes, in the form  `.text-*`, apply a color [token]([[docsref:/foundation/tokens]]) as `color` for a [content]([[docsref:/foundation/colors#content]]) or ['always' color]([[docsref:/foundation/colors#always]]) dynamically depending on the [color mode]([[docsref:/foundation/color-modes]]).

To colorize icons, please refer to our [icon color section](#on-icons).

OUDS Web provides many `.text-*` and `.bg-*` utilities, but they should be used with care to meet our design specifications and [WCAG 2.2 accessibility standards for color contrast](https://www.w3.org/TR/WCAG/#distinguishable).
To ensure compliance with the standard, define `color`, `background-color`, and `font-size` together.

<BrandSpecific brand="orange">
Thus, the `.text-brand-primary` color on light background (`#f15e00`) should only be used in a font size greater than 24px (using for example `.fs-ds` utility), or 19px bold (using for example `.fs-hm` and `.fw-bold` utilities).
The `.text-brand-primary` color on dark background (`#ff7900`) can be used in any size.
</BrandSpecific>

<Callout type="warning">
  Please note that aside from `.text-brand-primary`, `.text-default`, and `.text-muted`, most text-utility classes should be avoided as they are highly contextual. Instead, apply our [color modes]([[docsref:/foundation/color-modes]]) directly—as described in the [background utilities]([[docsref:/utilities/background]])—to get the right text color as well as ensure nested components render in the correct mode.
</Callout>

Here are some compliant combinations examples for texts:

<Example code={[...allTokens.filter(token => token.name.match(/\$ouds-color-content-/) && !token.name.match(/-action-/) && token.name.match(/-light/) && token.compiledValue).map(token => {
    if (token.name.match(/-on-/)) {
      return `<p class="bg-surface-${token.name.split('-').slice(4, -1).join('-')} text-${token.name.split('-').slice(3, -1).join('-')} p-small fs-hl">Text ${token.name.split('-').slice(3, -1).join('-')}</p>`
    }
    return `<p class="text-${token.name.split('-').slice(3, -1).join('-')} p-small fs-hl">Text ${token.name.split('-').slice(3, -1).join('-')}</p>`
  }),
  `<p class="text-always-black bg-always-white p-small fs-hl">Text always-black</p>
  <p class="text-always-white bg-always-black p-small fs-hl">Text always-white</p>
  <p class="text-always-on-black bg-always-black p-small fs-hl">Text always-on-black</p>
  <p class="text-always-on-white bg-always-white p-small fs-hl">Text always-on-white</p>`]} />

<Example code={[

<p class="text-${themeColor.name}-emphasis${themeColor.contrast_color ? ` bg-${themeColor.contrast_color}` : ''}">.text-${themeColor.name}-emphasis</p>`),
  `<p class="text-body">.text-body</p>
<p class="text-body-emphasis">.text-body-emphasis</p>
<p class="text-body-secondary">.text-body-secondary</p>

<p class="text-black-50">.text-black-50</p>
<p class="text-white-50">.text-white-50</p>`]} />

## Specificity

Sometimes contextual classes cannot be applied due to the specificity of another selector. In some cases, a sufficient workaround is to wrap your element’s content in a `<div>` or more semantic element with the desired class.
