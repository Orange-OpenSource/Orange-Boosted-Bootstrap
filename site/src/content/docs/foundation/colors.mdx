This page lists all the colors available in OUDS Web and their usage, organized by their category. All colors are displayed in both light and dark modes for reference. However, they should be applied semantically using their respective utility classes or custom properties to ensure dynamic adaptation to the [color mode]([[docsref:/foundation/color-modes]]).

## Background

These colors are used for whole pages and structurally meaningful sections backgrounds.

These colors can be used with [our background utilities]([[docsref:/utilities/background]]).

<Callout type="info" title="Usage reference">
  Please refer to the [background colors design reference](https://unified-design-system.orange.com/472794e18/p/217ac6-color/t/b3d025a22f) for more information on usage.
</Callout>

<ColorTable
  colorPrefix="ouds/color/bg/"
  tokenRegexp={/\$ouds-color-bg-(?<colorName>.*)-light/}
  getUsage={(colorName) => `.bg-${colorName}`} />

## Surface

These colors should be used as background for elements that are on another background.

These colors can be used with [our background utilities]([[docsref:/utilities/background]]). Some of these surface colors have an opacity value and are designed to come on top of another color, we explain [how to achieve surface color superposition here]([[docsref:/utilities/background#multiple-backgrounds]]).

<Callout type="info" title="Usage reference">
  Please refer to the [surface colors design reference](https://unified-design-system.orange.com/472794e18/p/217ac6-color/t/e89157f0cf) for more information on usage.
</Callout>

<ColorTable
  colorPrefix="ouds/color/surface/"
  tokenRegexp={/\$ouds-color-surface-((?!status|brand))(?<colorName>.*)-light/}
  getUsage={(colorName) => `.bg-surface-${colorName}`} />

<ColorTable
  colorPrefix="ouds/color/surface/brand/"
  tokenRegexp={/\$ouds-color-surface-brand-(?<colorName>.*)-light/}
  getUsage={(colorName) => `.bg-surface-brand-${colorName}`} />

### Status

These surface colors are used specifically to indicate a status.

Like other surface colors, they can be used with [our background utilities]([[docsref:/utilities/background]]).

{
  ['positive', 'info', 'warning', 'negative', 'accent'].map((status) => (
    <ColorTable
      colorPrefix={`ouds/color/surface/status/${status}/`}
      tokenRegexp={new RegExp(`\\$ouds-color-surface-status-${status}-\(\?<colorName>\.\*\)-light`)}
      getUsage={(colorName) => `.bg-surface-status-${status}-${colorName}`} />
  ))
}

## Always

These colors are invariant, they will stay the same on every brand, modes or contexts.

These colors can be used with [our background utilities]([[docsref:/utilities/background]]), [our text utilities]([[docsref:/utilities/color]]) or [our border utilities]([[docsref:/utilities/border]]).

<Callout type="info" title="Usage reference">

</Callout>

<ColorTable

## Opacity

These colors only define opacities levels, they are used to tweak visibility, hierarchy or emphasis of elements.

These colors can be used with [our opacity utilities]([[docsref:/utilities/opacity]]).

<Callout type="info" title="Usage reference">
  Please refer to the [opacity colors design reference](https://unified-design-system.orange.com/472794e18/p/217ac6-color/t/b4eb059054) for more information on usage.
</Callout>

<ColorTable
  colorPrefix="ouds/opacity/"
  tokenRegexp={/\$ouds-opacity-(?<colorName>.*)/}
  getUsage={(colorName) => `.opacity-${colorName}`}
  getStyle={(token, mode) =>
    mode === 'light'
      ? {opacity: token.lightValue, backgroundColor: 'var(--bs-color-surface-brand-primary)'}
      : {opacity: token.darkValue, backgroundColor: 'var(--bs-color-surface-brand-primary)'}} />
