For each types of items, OUDS Web has a common base `.item`, and `.item-main` classes that set up default styles such as padding and content alignment. The `.item` class is intended to be used in conjunction with our item variants, or to serve as a basis for your own custom styles. The most basic item is the `.item-navigation` variant.

### Approach

There are 2 main categories inside the items group of components:
- [Card items](#card-items) are single items that can be used as standalone components.
- [List items](#list-items) are designed to be used within a list, allowing multiple items to be displayed together.

Inside these categories, there are 2 sub-categories:
- Navigation items, which allow navigation.
- Static items, which are not interactive.

The following table summarizes the different types of items and their categories.

<BsTable>
  | Categories     | Card                 | List                 |
  | -------------- | -------------------- | -------------------- |
  | *Static*       | Static card item     | Static list item     |
  | *Navigation*   | Navigation card item | Navigation list item |
</BsTable>

The basic structure includes:
* `.item-wrapper` as the outermost container, which is used to add additional styles or behavior.
* `.item` as the container before the divider.
* `.item-main` as the main container for the primary content of the item.
* `.item-text-container` as the container for the text content of the item.
* `.item-interactive` as the class for interactive elements within the item, such as links or buttons. It's mandatory on the link for the [navigation item](#navigation-list-item).

## Card items

Card items are self-contained UI elements that can be used to present a group of related content. They can be used independently or within a card-based layout.

### [[comp]] Static card item

Static card item is a non-interactive UI element used to present a self-contained group or related content.
The card surface does not navigate, trigger an action or receive interactive state. When additional navigation is required, consider using a [navigation card item](#navigation-card-item) instead.

All the available variants can be found in the [common variants](#common-variants).

<Example buttonLabel="standard static items display" class="d-flex flex-column gap-medium" code={`
  <div class="item-wrapper">
    <div class="item">
      <div class="item-main">
        <div class="item-text-container">
          <p class="item-label">Label</p>
        </div>
      </div>
    </div>
  </div>`} />

### [[comp]] Navigation card item

A navigation card item is an interactive, self-contained card UI element that takes users to one internal or external destination.
Use it when navigational content requires stronger visual grouping or prominence than a standard [navigation list item](#navigation-list-item).

The card may be displayed independently or within a card-based layout and may use a distinct background, outline or rounded corners. The entire card acts as a single navigation target and supports the required hover, focus and pressed states. Do not use a Navigation card item when the card must contain several destinations or independent interactive controls.

Add a `.item-navigation` on the `.item-wrapper` level and an `.item-interactive` at the link level to create a navigation card item.

Available variants can be found in the [common variants section](#common-variants) and [navigation items variants section](#navigation-items-variants).

<Example buttonLabel="standard navigation item display" class="d-flex flex-column gap-medium" code={`<div class="item-wrapper item-navigation">
    <div class="item">
      <div class="item-main">
        <div class="item-text-container">
          <a href="#" class="item-label item-interactive">Label</a>
        </div>
      </div>
    </div>
  </div>`} />

Optionally, the `<a>` element can wrap the entire `.item` instead of just the label.

<Example buttonLabel="navigation item with link wrapping entire item" class="d-flex flex-column gap-medium" code={`<div class="item-wrapper item-navigation">
    <a href="#" class="item item-interactive">
      <div class="item-main">
        <div class="item-text-container">
          <p class="item-label">Label</p>
        </div>
      </div>
    </a>
  </div>`} />

## List items

The list items are UI elements used inside structured lists. They are surrounded by `.item-list` added to the `<ul>` element.

Some variants shouldn't be applied to this kind of items. Warning will be done about it in:
- [Outlined](#outlined) variants
<BrandSpecific brand={['orange', 'orange-compact']}>
- [Rounded](#rounded-corners) variant
</BrandSpecific>

### [[comp]] Static list item

A static list item is a non-interactive horizontal row UI element used to present a unit of information within a structured list.
It supports quick scanning and comparison between related items and may include supporting text, leading or trailing content, and status information. It does not navigate, trigger an action, or receive interactive states.

All the available variants can be found in the [common variants section](#common-variants).

<Example buttonLabel="standard static items display" class="d-flex flex-column gap-medium" code={`<ul class="item-list">
    <li class="item-wrapper">
      <div class="item">
        <div class="item-main">
          <div class="item-text-container">
            <p class="item-label">Label</p>
          </div>
        </div>
      </div>
    </li>
    <li class="item-wrapper">
      <div class="item">
        <div class="item-main">
          <div class="item-text-container">
            <p class="item-label">Label</p>
          </div>
        </div>
      </div>
    </li>
  </ul>`} />

### [[comp]] Navigation list item

A navigation list item is an interactive horizontal row UI element that takes users to another internal or external destination
when activated.

It is intended for structured lists where users need to scan and access related destinations quickly. The entire item acts as a single navigation target and supports the required hover, pressed and keyboard focus states.

Add a `.item-navigation` on the `.item-wrapper` level and an `.item-interactive` at the link level.

Available variants can be found in the [common variants section](#common-variants) and [navigation items variants section](#navigation-items-variants).

<Example buttonLabel="standard navigation item display" class="d-flex flex-column gap-medium" code={`<ul class="item-list">
    <li class="item-wrapper item-navigation">
      <div class="item">
        <div class="item-main">
          <div class="item-text-container">
            <a href="#" class="item-label item-interactive">Label</a>
          </div>
        </div>
      </div>
    </li>
    <li class="item-wrapper item-navigation">
      <div class="item">
        <div class="item-main">
          <div class="item-text-container">
            <a href="#" class="item-label item-interactive">Label</a>
          </div>
        </div>
      </div>
    </li>
  </ul>`} />

## Navigation items variants

These variants are exclusive to the navigation items. They are used to indicate a specific type of navigation. The other common variants can be found in the [common variants section](#common-variants).

### Back chevron

Add `.item-previous` to the `.item-navigation` to indicate that the item is a back navigation item. This will add a back chevron icon to the start of the item.

<Example buttonLabel="navigation item with back chevron" class="d-flex flex-column gap-medium" code={`<div class="item-wrapper item-navigation item-previous">
    <div class="item">
      <div class="item-main">
        <div class="item-text-container">
          <a href="#" class="item-label item-interactive">Label</a>
        </div>
      </div>
    </div>
  </div>`} />

### External link

Add `.item-external` to the `.item-navigation` to indicate that the item is an external link. This will add an external link icon to the end of the item.

<Example buttonLabel="navigation item with external link" class="d-flex flex-column gap-medium" code={`<div class="item-wrapper item-navigation item-external">
    <div class="item">
      <div class="item-main">
        <div class="item-text-container">
          <a href="#" class="item-label item-interactive">Label</a>
        </div>
      </div>
    </div>
  </div>`} />

## Common variants

These variants can be used in all types of items.

### With asset

The main container `.item-main` of an item can contain up to 3 elements. They always appear in the following order:
1. [Leading container](#leading-container) using `.item-leading-container` (optional)
2. [Text container](#texts) using `.item-text-container` (required)
3. [Trailing container](#trailing-container) using `.item-trailing-container` (optional)

#### Texts

There are 4 text elements that are meant to be used in the `.item-text-container`.

They always appear in the following order:
1. Overline using `.item-overline` (optional) (only if not [small item](#sizes))
2. Label using `.item-label`, that can optionally be bold (using `.fw-bold`) (required)
3. Extra label using `.item-extra-label` (optional) (only if not [small item](#sizes))
4. Description using `.item-description` (optional)

Optionally, a slot can be added at the end of the text container using `.item-slot` and/or in place of the `.item-label` if it replaces it.

<Example buttonLabel="item text" class="d-flex flex-column gap-medium" code={`<div class="item-wrapper">

      <div class="item-main">
        <div class="item-text-container">
          <p class="item-overline">Overline</p>
          <p class="item-label">Label</p>
          <p class="item-extra-label">Extra label</p>
          <p class="item-description">Description</p>
        </div>
      </div>
    </div>
  </div>

  <!-- Label bold -->

    <div class="item">

#### Slots

This component contains many slots that can be used to add custom content. The main slots are located in the [leading container](#leading-container), the [text container](#texts), and the [trailing container](#trailing-container). The slots are intended to be used for custom content that is not covered by the other elements of the component. We can add another slot outside of the `.item-main` container as well.

<Callout type="warning" title="Small list item">
  The slots shouldn't be used in a [small list item](#sizes).
</Callout>

Here is a small resume of the slots that can be used in a static item:
* [Leading container](#leading-container) using `.item-slot` (optional)
* [Text container](#texts) using `.item-slot` (optional)
* [Trailing container](#trailing-container) using `.item-slot` (optional)
* A slot outside of the `.item-main` container, direct child of `.item` using `.item-slot` (optional)

Optionally, if the item is a [navigation item](#navigation-item) and a slot is also interactive, you can add `.item-slot-clickable` to the slot.<BrandSpecific brand={['orange', 'orange-compact']}> Please note that the slot won't be [rounded](#rounded-corners) anymore.</BrandSpecific>

<Example buttonLabel="item using a slot outside of the main content of the item" class="d-flex flex-column gap-medium" code={`<div class="item-wrapper">

      <div class="item-main">
        <div class="item-text-container">

      <div class="item-slot">

## States

### Disabled

Items can be disabled by using `.disabled` on the element `.item-wrapper` or any ancestors. The disabled style is also applied to all interactive elements inside the item.

If the item is in a disabled context, the item will automatically take the disabled state without needing to add the `.disabled` class. This works if the ancestor element has the `[disabled]` attribute, the `[aria-disabled="true"]` attribute, or the `.disabled` class.

You may handle on your own the [slots](#slots) regarding accessibility and design. For example, if you have a slot with a button, you may want to disable the button to apply the right behavior and the correct visual state.

<Callout type="info">
  The images and videos inside the disabled static item will be grayed out using opacity. You'll still need to handle it correctly from an accessibility point of view.
</Callout>
