## Tag

Tag indicator can be used near some text (to show status or key details), or in a list (to indicate or filter by categories or keywords).

<Callout type="info">
  For accessibility and semantics reasons, tags, like any other informative elements, must be enclosed in an ancestor or parent semantic tag. In the following examples, tag are used either in `<li>`, `<h2>` or `<p>`: you can use any other semantic tag depending on the usage context.
</Callout>

OUDS Web has a basic `.tag` class that sets up default styles.

<Example code={`<ul class="list-unstyled d-flex column-gap-xs">
    <li><span class="tag">Roaming</span></li> <!-- Issue with line-height of li -->
    <li><span class="tag">5G Ready</span></li>
    <li><span class="tag">4G</span></li>
  </ul>
  <h2>Title <span class="tag">Coming soon</span></h2>
  <p><span class="tag">New</span> Some text</p>`
  } />

To display tags with squared corners, add the `.rounded-none` utility class to the tag.

<Example code={`<ul class="list-unstyled d-flex flex-wrap column-gap-xs row-gap-md">

    <li><span class="tag rounded-none tag-disabled">Disabled</span></li>
  </ul>`} />

To add an asset (either bullet, icon or loader) inside a tag, add a `.tag-asset-container` before the tag's text.

## Tag input

<Callout type="warning">
  Please note that all the variants above shouldn't be applied on this version of the component.
</Callout>

Standard tags are used in an informative way. Tag inputs are interactive and can take all the usual states: enabled, hover, pressed, disabled or focused. The tag input should be used along with a `<button>`.

They aren't removed on click at the moment, so you need to implement this behavior by yourself.

    <li><button type="button" class="tag tag-input">Label</button></li>
    <li><button type="button" class="tag tag-input">Label</button></li>
  </ul>`} />
