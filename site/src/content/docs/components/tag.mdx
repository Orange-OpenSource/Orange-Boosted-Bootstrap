## Tag (indicator)

<Callout type="info">
  For accessibility and semantics reasons, tags, like any other informative elements, must be on or be enclosed in a semantic HTML tag. In the following examples, tag are used either as `<li>` or in `<h2>` or `<p>`: you can use any other semantic tag depending on the usage context.
</Callout>

<Example code={`<ul class="list-unstyled d-flex column-gap-xs">
    <li class="tag">Roaming</li>
    <li class="tag">5G Ready</li>
    <li class="tag">4G</li>
  </ul>`
} />

OUDS Web [color and background utilities]([[docsref:/helpers/color-background]]) can be applied to display colored tag variants, either emphasized or muted. Each color has its own semantic purpose. Please follow the [tag design specifications](https://unified-design-system.orange.com/) to choose the right color for the right semantic. {/* TODO: set the right url for design documentation */}

For disabled color, use the `.tag-disabled` class.

<Callout type="warning">
  The disabled tag should only be used in a disabled context, inside an interactive element that has been disabled, such as a card.
</Callout>

### Variants

#### Squared

To display tags with squared corners, add the `.rounded-none` utility class to the tag.

<Example code={`<ul class="list-unstyled d-flex flex-wrap column-gap-xs row-gap-md">
    <li class="tag rounded-none text-bg-status-accent-emphasized">Accent emphasized</li>
    <li class="tag rounded-none text-bg-status-accent-muted">Accent muted</li>
    <li class="tag rounded-none tag-disabled">Disabled</li>
  </ul>`} />

#### With asset

To add an asset (either bullet, icon or loader) inside a tag, add a `.tag-asset-container` before the tag's text.

### Layout

Tag indicator can be used near some text or some title (to show status or key details), or in a list (to indicate categories or keywords). If needed, you can use the [vertical alignment utilities]([[docsref:/utilities/vertical-align]]) to vertically center inline content.

<Example code={`<ul class="list-unstyled d-flex column-gap-xs">
    <li class="tag">Roaming</li>
    <li class="tag">5G Ready</li>
    <li class="tag">4G</li>
  </ul>
  <ul class="list-unstyled d-flex column-gap-xs">
    <li class="tag tag-sm">Roaming</li>
    <li class="tag tag-sm">5G Ready</li>
  </ul>
  <h2>Title <span class="tag align-middle">Coming soon</span></h2>
  <h2>Title <span class="tag tag-sm align-middle">Coming soon</span></h2>
  <p><span class="tag">New</span> Some text</p>
  <p><span class="tag tag-sm">New</span> Some text</p>`} />

Tags can be added side by side. They require horizontal and vertical spacing between each other. You can use [flexbox utilities]([[docsref:/utilities/spacing#gap]]) to manage the horizontal space (default value is obtained thanks to `.column-gap-xs`) the and vertical space (default value is obtained thanks to `.row-gap-md`).

<Example code={`<ul class="list-unstyled d-flex flex-wrap col-4 column-gap-xs row-gap-md">
    <li class="tag">Label</li>
    <li class="tag">Label</li>
    <li class="tag">Label</li>
    <li class="tag">Label</li>
    <li class="tag">Label</li>
    <li class="tag">Label</li>
  </ul>`} />

## Tag input

<Callout type="warning">
  Please note that all the variants of tag (indicator) above must not be applied on tag input.
</Callout>

Tag inputs are interactive and may be associated to an input field to be added dynamically. The tag input should be used along with a `<button>`, and can take all the usual states: enabled, hover, pressed, disabled or focused.

It can be removed on click, but you have to implement this behavior by yourself, with some JavaScript specific to your project.

  <Example code={`<ul class="list-unstyled d-flex flex-wrap column-gap-xs">
    <li><button type="button" class="tag tag-input">Roaming</button></li>
    <li><button type="button" class="tag tag-input">5G Ready</button></li>
    <li><button type="button" class="tag tag-input">4G</button></li>
  </ul>`} />

  ### Layout


  Tag inputs are typically intended to be added side by side. They require horizontal and vertical spacing between each other. You can use flexbox utilities to manage the horizontal space (default value is obtained thanks to `.column-gap-xs`). Vertical space is automatically created by the extended touch target heights.

  <Example code={`<ul class="list-unstyled d-flex flex-wrap col-4 column-gap-xs">
    <li><button type="button" class="tag tag-input">Label</button></li>
    <li><button type="button" class="tag tag-input">Label</button></li>
    <li><button type="button" class="tag tag-input">Label</button></li>
    <li><button type="button" class="tag tag-input">Label</button></li>
    <li><button type="button" class="tag tag-input">Label</button></li>
    <li><button type="button" class="tag tag-input">Label</button></li>
  </ul>`} />

  ### States

  #### Disabled

  To disable a tag input, just add the `disabled` attribute to the `.tag`.

  <Example code={`<ul class="list-unstyled d-flex flex-wrap column-gap-xs">
    <li><button type="button" class="tag tag-input" disabled>Roaming</button></li>
    <li><button type="button" class="tag tag-input">5G Ready</button></li>
    <li><button type="button" class="tag tag-input" disabled>4G</button></li>
  </ul>`} />
