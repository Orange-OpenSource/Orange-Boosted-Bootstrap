## Overview

Modal is built on the native `<dialog>` element and opened/closed declaratively with invoker commands.

Use:
- `commandfor="dialog-id" command="show-modal"` to open.
- `commandfor="dialog-id" command="close"` to close.

## Modal dialog

For simple dialog with few content, for example confirmation dialogs, use the `modal-dialog` class.

<Example buttonLabel="dialog modal basic" code={`<dialog id="demoModalDialog" class="modal modal-dialog" aria-label="Demo modal dialog">
    <div class="modal-header">
      <button type="button" class="btn btn-close" commandfor="demoModalDialog" command="close">
        <span class="visually-hidden">Close</span>
      </button>
    </div>
    <div class="modal-body">
      <h4>Are you sure you want to delete this item?</h4>
    </div>
    <div class="modal-footer">
      <button type="button" class="btn btn-default" commandfor="demoModalDialog" command="close">Cancel</button>
      <button type="button" class="btn btn-negative" commandfor="demoModalDialog" command="close">Delete</button>
    </div>
  </dialog>

  <button type="button" class="btn btn-default" commandfor="demoModalDialog" command="show-modal">
    Open modal
  </button>`} />

## Accessibility

- Provide an accessible name with `aria-labelledby` or `aria-label`.
- Keep a visible close control.
- Native dialog handles Escape and focus behavior by default.

## Notes

- Use `data-bs-role="modal"` on dialogs styled as OUDS modal.
- Use `data-bs-backdrop-static` on the dialog to prevent backdrop click dismissal.
